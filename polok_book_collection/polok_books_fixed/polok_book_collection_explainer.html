Polok Book Collection — Full Project Overview

PHP + Text file দিয়ে তৈরি একটি personal book library management system · A personal book library built without any database

এই project কীভাবে কাজ করে? / How does it work?
এখানে কোনো MySQL বা database নেই। সমস্ত data দুটো plain text file এ pipe (|) দিয়ে আলাদা করে সংরক্ষিত হয়। PHP session দিয়ে login ও logout manage করা হয়।

No database is used. All data is stored in two plain text files, with fields separated by a pipe (|) character. Login and logout are managed using PHP sessions.

login.php — Session-based authentication

user.txt পড়ে username + password মিলিয়ে session শুরু করা হয় · Reads user.txt, matches credentials, and starts a session

Session start at top of page / Page এর একদম শুরুতে session চালু করা
// session_status() চেক — already started কিনা দেখা // Avoid starting session twice if(session_status() === PHP_SESSION_NONE) { session_start(); } // আগে থেকে logged in থাকলে সরাসরি main page এ পাঠিয়ে দাও // If already logged in, skip login page entirely if(isset($_SESSION['user_name'])) { header("location:main.php"); exit(); }
Login logic — match user from file / user.txt থেকে username ও password মেলানো
$user_file = __DIR__ . "/user.txt"; if(file_exists($user_file)) { // সব line array তে লোড করা / Load all lines into array $lines = file($user_file, FILE_IGNORE_NEW_LINES | FILE_SKIP_EMPTY_LINES); foreach($lines as $line) { $parts = explode("|", $line); if(count($parts) < 7) continue; // ভাঙা line skip করা // parts[6]=username, parts[3]=password if($parts[6] === $input_user && $parts[3] === $input_pass) { $_SESSION["user_name"] = $parts[6]; $_SESSION["full_name"] = $parts[1]; // header এ নাম দেখাবে header("location:main.php"); exit(); } } } $error = 'Invalid username or password.';
Session কীভাবে কাজ করে? / How does a session work?
Login সফল হলে server এ একটি session file তৈরি হয় এবং browser এ একটি cookie পাঠানো হয়। পরের প্রতিটি request এ browser সেই cookie পাঠায়, server session file থেকে user চেনে। Logout করলে session destroy হয়ে যায়।

On successful login, PHP creates a session file on the server and sends a cookie to the browser. On every subsequent request, the browser sends the cookie back, and the server identifies the user from the session. Logout destroys the session.

Bugs Fixed — কী ভুল ছিল, কীভাবে ঠিক হলো

আগের version এ ৬টি bug ছিল · The previous version had 6 bugs — here's each one explained

BUG 1 Relative file path — সব ফাইলে
সমস্যা / Problem:
fopen("user.txt") লেখা ছিল। এটি server এর current working directory তে file খোঁজে, যা project folder থেকে আলাদা হতে পারে। ফলে file পাওয়া যেত না।

fopen("user.txt") looks in the server's current working directory, which may not be the project folder — causing the file to go missing.
❌ ভুল / Wrong
fopen("user.txt", "a")
✅ সঠিক / Correct
fopen(__DIR__ . "/user.txt", "a")
BUG 2 Duplicate user check ছিল না
সমস্যা / Problem:
Same username বা ID দিয়ে বারবার register করা যাচ্ছিল। Login এ wrong user match হওয়ার সম্ভাবনা ছিল।

Users could register multiple times with the same username or ID, potentially causing wrong login matches.
সমাধান / Fix:
Registration এর আগে user.txt এর সব line পড়ে check করা হয় — same ID, username বা email পাওয়া গেলে error দেওয়া হয় এবং save হয় না।

Before saving, all existing lines are checked. If the same ID, username, or email is found, registration is blocked with an error.
BUG 3 Already logged in redirect নেই
সমস্যা / Problem:
Login থাকা অবস্থায় login.php এ গেলে আবার form দেখাচ্ছিল — এটি confusing ও unnecessary।

Visiting login.php while already logged in showed the login form again — confusing UX.
❌ ভুল / Wrong
// কোনো check ছিল না session_start(); // form directly shown
✅ সঠিক / Correct
session_start(); if(isset($_SESSION['user_name'])){ header("location:main.php"); exit(); }
BUG 4 Session check ভুল — header.php
সমস্যা / Problem:
if(!isset($_SESSION)) — PHP তে $_SESSION সবসময় defined থাকে, তাই এই condition কখনো true হয় না। Session কখনো properly start হচ্ছিল না।

In PHP, $_SESSION is always defined, so !isset($_SESSION) is never true — the session was never being started correctly.
❌ ভুল / Wrong
if(!isset($_SESSION)) session_start(); // never runs!
✅ সঠিক / Correct
if(session_status() === PHP_SESSION_NONE) session_start();
BUG 5 empty check logic ভুল — manage_books.php
সমস্যা / Problem:
empty($lines) চেক করার পরেও নিচের foreach loop চলছিল। দুটো আলাদা if block এ ছিল না বলে এটি হচ্ছিল।

The foreach ran even when $lines was empty, because the empty check and the loop weren't in an if/else structure.
সমাধান / Fix:
if(empty($lines)){...} else { foreach... } — এইভাবে if/else দিয়ে ঠিক করা হয়েছে। এখন empty হলে foreach কখনো চলে না।

Wrapping in if/else ensures the foreach only runs when data exists.
BUG 6 Trailing empty line জমছিল — delete/edit এ
সমস্যা / Problem:
বারবার edit বা delete করলে books_info.txt এ ফাঁকা line জমে যাচ্ছিল। Empty array হলেও "\n" লেখা হচ্ছিল, যা parse করতে গিয়ে extra empty entry তৈরি করছিল।

After repeated edits/deletes, blank lines accumulated in the file. Even empty data was written as a newline, creating phantom empty entries on the next parse.
❌ ভুল / Wrong
file_put_contents($path, implode(PHP_EOL,$data).PHP_EOL); // empty array হলেও "\n" লেখে
✅ সঠিক / Correct
$content = empty($data) ? '' : implode(PHP_EOL,$data).PHP_EOL; file_put_contents($path, $content);
